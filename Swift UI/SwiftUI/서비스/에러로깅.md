# 에러 로깅

서버 통신(NetWorking) 과정에서 오류가 생길때 

* 데이터가 오가는게 네트워킹이라고 하는 것

* 서버통신은 서버에서 데이터가 오가는게 네트워킹이다 

xcode&VScode들을 IDE라고 한다 

데이터가 출발했을때 도착에서의 결과가 맞냐 안맞냐로 bool로 표현한다 (True, False)

디버깅 중요

서버에서 에러다 아니다를 지정해준다 "숫자로"

서버에서 로직을 짜고 클라이언트에서 응답을 하면 응답의 값이 데이터고 네트워킹을 하면 서버에서 필요로 하는 응답 값과 일치 한지 아닌지 보려면 if를 쓴다 일치하지 않으면 "에러" 를 표시 
```swift
클라이언트 문제이면 무조건 "4"를 표시 4는 클라이언트 문제임 응답 문제 

서버 문제이면 "5" 를 표시 5는 서버의 문제이다 요청문제

둘이 연결이 잘되면 "2"를 보내준다 

409는 예외로 클라이언트랑 서버 문제가 아닌  중복 이라는 뜻 임 DB에 있는 값이 겹친 것임 
```

서버통신이 느려지는 이유가 클라이언트랑 서버랑 에러 로깅을 통해서 맞춰가는건데 서버나 클리이언트에서 잘못하면 계속 늘어난다
