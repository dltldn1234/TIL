# service

-특정 기능을 대신 처리해주는 역할-

예를 들면 

로그인 해주는 역할
```swift
LoginService
```
서버 통신 해주는 역할
```swift
NetWorkService
```
데이터 저장해주는 역할
```swift
StorageService
```
이런 걸 전부다 서비스라고 말한다


# 서비스(Service)가 왜 필요하냐?

* 코드를 역할별로 나눠서 관리하기 위해서


만약 서비스 없이 만들면?

예를 들어 ViewController 안에

서버 통신
데이터 저장
로그인
UI 처리

전부 다 넣는다고 생각해보셈.

class ViewController: UIViewController {

    func login() {

        // 서버 통신
        print("서버 요청")

        // 데이터 저장
        print("토큰 저장")

        // 화면 변경
        print("메인 화면 이동")
    }
}

이러면 코드가 길어지고 읽기가 어렵고수정 힘들고 오류 찾기가 어렵기 때문에

그래서 Service를 나누는 것이다

*예시*
```swift
View → 화면 담당
Service → 기능 담당
Model → 데이터 담당
```

```swift
//로그인 담당
class LoginService {

    func login() {
        print("서버 로그인 요청")
    }
}
```

```swift
//데이터 저장
class StorageService {

    func save() {
        print("데이터 저장")
    }
}
```

## 한문장으로 정의 하면 코드를 기능별로 분리해서 관리하기 쉽게 만들어주는 것 이다
