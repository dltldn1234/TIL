# 동기와 비동기

## 동기란?
우리가 로그인을하면 서버의 응답을 받아야 원하는 페이지로 가는거임
서버의 응답을 받으면 다음으로 넘어가는것이 동기 
토큰을 받기위해 로그인과 회원가입을 하는 것


## 비동기란?
서버에서 응답을 안해도 UI가 알아서 먼저 맞는지 안맞는지 검사안하고
띄어주는것 그리고 띄어주고 나서 서버가 응답을해서 맞으면 그대로 아니면 다시 로그인 화면으로 가는거 
__________ 
통신-> 뷰생성

## 앱 처리 방식

* 앱의 전체적인 화면은 비동기로 처리해야된다 
     * Ui가 빠르기때문에 서버통신이 연결되기 전에 UI부터 띄우고 그 띄우는 순간에 서버랑 연결이되면서 이루어 지기 때문이다

### 스켈레톤

* 아무것도 없는 상태를 스켈레톤이라고한다 서버통신을 하지않는 뼈만 남은 상태

```swift
동기와 비동기 구조

동기는 직렬구조

비동기는 병령 구조

비동기는 동시에 

동기는 동시에가 안됨
```
