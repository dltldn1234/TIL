# 낙천적 프로그래밍
-이름 대로 정해진 규격이 없이 프로그래밍을 한다는 뜻-
>* 사용자가 수행한 동작이 성공할 것이라고 "낙천적"으로 가정하고, 서버의 응답을 기다리지 않고 화면을 즉시 업데이트하는 기술을 의미함

비동기와 비슷한데 비동기는 로직이고 낙천적 프로그래밍은 UI관련이다


```swift
ex) 먼저 UI가 켜지긴 켜지는데 데이터가 들어오기 전까지 기다리는 것(카카오맵을 예시로 하면 무슨 장소를 누르고 그에대한 세부정보가 있는 창은 데이터가 전부다 들어온다음에 뜨는 것 이다)
```

**작업할게 없으면 nil로 보내는데 비동기는 오류로 보내버린다 \
(작업할게 있다는 것이 스켈리톤화면이 보여지는 것 )**

인스타 게시물애 좋아요를 눌렀는데 버그 때메 누르기 전으로 돌아가는게 비동기이다 


# 낙천적 프로그래밍의 반대 방어적 프로그래밍
-문제가 생길 수도 있다고 생각하고 미리 검사하는 방식-
* 예상치 못한 입력, 잘못된 상태, 또는 외부 요인에 의해 프로그램이 비정상적으로 종료(Crash)되거나 오작동하는 것을 미연에 방지하는 설계 기법


1. Swift 방어적 프로그래밍의 핵심 요소옵셔널(Optional) (?, !, if let, guard let): 값이 없거나 nil일 수 있는 상황을 컴파일 타임에 강제 처리하여 안전성을 확보한다

Guard 구문 (guard): 함수의 조기 종료(Early Exit)를 통해 잘못된 입력이나 조건에 대해 안전한 코드 블록을 제공을 함

타입 안전성(Type Safety): 엄격한 타입 통제로 잘못된 데이터 타입 사용을 막는다

에러 처리(Error Handling) (do-catch, try): 발생 가능한 에러를 예외 처리하여 프로그램이 계속 기능을 수행할 수 있게 한다

Type Checking & Casting (as?, as!): 타입 변환 시 안전하게 처리함
