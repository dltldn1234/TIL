# 클래스
**데이터(프로퍼티)와 기능(메서드)을 같이 묶는 타입(참조타입)**

*정의*

```swift
class 이름{
     /* 구현부 */
}
```
```swift
class Sample {
    var mutableProperty: Int = 100     // 가변 프로퍼티
    let immutableProperty: Int = 100   // 불변 프로퍼티

    static var typeProperty: Int = 100 // 타입 프로퍼티

    // 인스턴스 메서드
    func instanceMethod() {
        print("instance method")
    }

    // 타입 메서드
    // 재정의 불가 타입 메서드 - static
    static func typeMethod() {
        print("type method - static")
    }

    // 재정의 가능 타입 메서드 - class
    static func typeMethod() {
        print("type method - class")
    }
}
```
 **학생 구조체 예시 **
```swift
class Student {
    var name: String = "unknown"
    var `class`: String = "Swift" 

    static func selfIntroduce() { // 타입 메서드
        print("학생타입입니다")
    }

    func selfIntroduce() { // 인스턴스 메서드
        print("저는 \(self.class)반 \(name)입니다")
    }
}
```

### 이니셜라이저 (Initializer)
**객체가 만들어질 때 실행되는 함수**

 * 객체 생성할 때 실행
 ```swift
 class Person {
    var name: String

    init(name: String) { // 생성
        self.name = name
    }
}

let p = Person(name: "yagom")
 ```

 ### 디이니셜라이저 (Deinitializer)
**객체가 메모리에서 사라질 때 실행되는 함수**
* 객체 메모리에서 사라질 때 실행
```swift
class Person {
    var name: String

    init(name: String) {
        self.name = name
    }

    deinit { // 제거
        print("메모리 해제됨")
    }
}

```

## 핵심 역할

1. 데이터 + 기능 묶기

2. 객체(인스턴스) 생성

3. 상속가능( 핵심 특징)
    
* 다종상속(한 클래스가 여러 부모 클래스를 동시에 상속받는것을 말함)이 안됨
    
4. 참조 타입(값 복사가 아니라 주소 공유임)
    
- 같은 객체를 같이 쓰는 느낌(연결되있음)
    
5. 상태 관리에 좋음 
    
- 앱에서 많이 쓰는 이유
    
    - 사용자 정보
    - 게임 캐릭터 상태
    - 화면(viewController)

“변하는 상태” 관리할때  클래스를 씀

### 클래스= 상태+ 행동 + 상속 + 참조형 객체
