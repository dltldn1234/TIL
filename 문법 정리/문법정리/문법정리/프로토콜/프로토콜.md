# 프로토콜
* 이 기능은 꼭 구현해야 한다 라고 규칙을 정해놓는 것

프로토콜은 특정 역할을 수행하기 위한 메서드, 프로퍼티, 이니셜라이저 등의 요구사항을 정의함

구조체, 클래스, 열거형은 프로토콜을 채택해서 프로토콜의 요구사항을 실제로 구현 가능

어떤 프로토콜의 요구사항을 모두 따르는 타입은 그’프로토콜을 준수한다’고 표현함

프로토콜의 요구사항을 충족시키려면 프로토콜이 제시하는 기능을 모두 구현해야됨

## 프로토콜 채택 및 준수
읽기전용 프로퍼티 요구는 연산 프로퍼티로 대체가 가능함
```swift
struct Person {
    var firstName: String
    var lastName: String
    
    var fullName: String {   // 연산 프로퍼티 (읽기 전용)
        return firstName + lastName
    }
}
```
읽기, 쓰기 프로퍼티도 연산 프로퍼티로 대체 가능

```swift
struct User {
    var firstName: String
    var lastName: String
    
    var fullName: String {   // 연산 프로퍼티 (읽기, 쓰기)
        get {
            return firstName + lastName
        }
        set {
            firstName = newValue
            lastName = ""
        }
    }
}
```

## 프로토콜 상속

프로토콜은 클래스와 다르게 다중상속이 가능함
```swift
protocol Animal {
    func eat()
}

protocol Dog: Animal {
    func bark()
}
```


### 클래스 상속과 프로토콜

클래스에서 상속과 프로토콜 채택을 동시에 하려면 상속받으려는 클래스를 먼저 명시하고 

그 뒤에 채택할 프로토콜 목록을 작성함
```swift
class SuperClass: Readable {
    func read() {
        print("read")
    }
}

class SubClass: SuperClass, Writeable, ReadSpeakable {
    func write() {
        print("write")
    }
    
    func speak() {
        print("speak")
    }
}
```

### 프로토콜 준수 확인

인스턴스가 특정 프로토콜을 준수하는지 확인할 수있음

is, as 연산자 사용
```swift
let sup: SuperClass = SuperClass()
let sub: SubClass = SubClass()

var someAny: Any = sup
someAny is Readable        // true
someAny is ReadSpeakable   // false
```
