# 오류처리
* 프로그램에서 발생할 수 있는 오류를 안전하게 처리하는 방법

## 오류표현

프로토콜과 (주로) 열거형을 통해서 오류를 표현함
```swift
enum 오류종류이름: Error {
    case 종류1
    case 종류2
    case 종류3
    // ...
}
```


오류 발생의 여지가 있는 메서드는 throws를 사용하여 오류를 내포하는 함수임을 표시함


**오류처리**

오류발생의 여지가 있는 throws 함수(메서드)는
 try를 사용하여 호출해야함

 try, try?, try!

 ### do- catch

 오류발생의 여지가 있는 throws 함수( 메서드)는
  do-catch 구문을 활용하여

 오류발생에 대비한다

 # try?와 try!

 ### try?

별도의 오류처리 결과를 통보받지 않고

오류가 발생했으면 결과값을 nil로 돌려받을 수 있음 정상동작 후에는 옵셔널 타입으로 정상 반환값을 돌려받음
```swift

enum MyError: Error {
    case fail
}

func test() throws -> Int {
    throw MyError.fail
}

let result = try? test()
print(result) // nil
```


### try!

오류가 발생하지 않을 것이라는 강력한 확신을 가질 때 

try!를 사용하면 정상동작 후에 바로 결과값을 돌려받는다

오류가 발생하면 런타임 오류(코드 실행 중에 발생하는 오류)가 발생하여 애플리케이션 동작이 중지됨
```swift
func test2() throws -> Int {
    return 10
}

let value = try! test2()
print(value) // 10
```
